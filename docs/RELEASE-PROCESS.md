# قاعدة العمل الإلزامية — الإصدار وسجل التغييرات (الدستور)

> **إلزامية**: مع كل موجة تغيير مكتملة يجب تنفيذ الخطوتين التاليتين معاً، وإلا يُعتبر العمل ناقصاً.

## 1) رفع رقم الإصدار
عدّل **مكاناً واحداً فقط**: [`AppVersion.cs`](../src/MarkdownStudio/AppVersion.cs)
- `AppVersion.Current` — بصياغة SemVer:
  - **MINOR + تصفير PATCH** (الافتراضي) لكل موجة ميزات مكتملة. مثال: `1.0.0 → 1.1.0`.
  - **MAJOR** عند كسر التوافق. مثال: `1.4.0 → 2.0.0`.
  - **PATCH** لإصلاح صغير فقط. مثال: `1.1.0 → 1.1.1`.
- `AppVersion.ReleasedDate` — تاريخ الإصدار (ISO: `YYYY-MM-DD`).

## 2) إضافة مدخلة في CHANGELOG
أضف مدخلة **في الأعلى** من [`CHANGELOG.md`](../src/MarkdownStudio/CHANGELOG.md) بنفس البنية حرفياً
(المحلّل هشّ تجاهها عمداً ليبقى بسيطاً — لا تغيّر الأنماط):

```
## [x.y.z] - YYYY-MM-DD
### HIGHLIGHTS
- بند بارز
### NEW
- جديد
### IMPROVED
- تحسين
### FIXED
- إصلاح
```

قواعد الصياغة:
- ترويسة النسخة: بالضبط `## [الإصدار] - التاريخ`.
- الأقسام المسموحة فقط: `HIGHLIGHTS` · `NEW` · `IMPROVED` · `FIXED` — كلّها اختيارية، **احذف الفارغ**.
- كل بند سطر يبدأ بـ `- `. يدعم `**عريض**` بسيط.
- الأحدث في الأعلى.

## القيود (ممنوعات)
- ❌ كتابة رقم إصدار حرفياً خارج `AppVersion` (بما فيها `.csproj` — أُزيل منه عمداً).
- ❌ كسر أنماط CHANGELOG (الترويسة/الأقسام/البنود).
- ✅ الأرقام تُعرض لاتينيّةً دائماً حتى في الواجهة العربية (RTL)؛ الاتجاه يتبع اللغة.

## لماذا؟
لوحة «ما الجديد» تقرأ `CHANGELOG.md` **وقت التشغيل** (مورد مضمّن + احتياط قرص)، وتظهر
تلقائياً مرّة واحدة عند تغيّر `AppVersion.Current` (مقارنةً بـ `LastShownWhatsNewVersion` في الإعدادات).
تطبيق الخطوتين يبقي اللوحة متزامنة مع الكود بلا خطوة يدوية منفصلة.
